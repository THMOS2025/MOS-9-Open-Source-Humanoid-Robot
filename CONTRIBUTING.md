# Contributing to MOS9 Open Source Hub

Thank you for helping improve MOS9.

## Scope

This repository is a coordination hub. Please use it for:

- Repository map updates
- Cross-repository documentation updates
- Onboarding and contributor workflow improvements

Feature implementation should usually happen in the appropriate sub-repository.

## Pull Request Guidelines

1. Create a focused branch for one change.
2. Keep changes small and clearly scoped.
3. Update `README.md` and `repos.yaml` together if repository metadata changes.
4. Use clear commit messages.
5. Open a pull request with context, motivation, and testing notes.

## Reporting Issues

- Use bug reports for broken links, stale repo mappings, or workflow problems.
- Use feature requests for hub-level improvements.

## Code of Conduct

Please follow [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
